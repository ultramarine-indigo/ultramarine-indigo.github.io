    PEFT(Parameter-Efficient Fine-Tuning)方法总结

    发表于 2024-02-18 分类于 论文详解

    PEFT 技术旨在通过最小化微调参数的数量和计算复杂度，来提高预训练模型在新任务上的性能，从而缓解大型预训练模型的训练成本。这样一来，即使计算资源受限，也可以利用预训练模型的知识来迅速适应新任务，实现高效的迁移学习。因此，PEFT 技术可以在提高模型效果的同时，大大缩短模型训练时间和计算成本。

    PEFT方法分类

    如上图所示，PEFT技术可大致分为四个大类：

    • Additive PEFT ：通过注入新的可训练模块或参数修改模型架构。
    • Selective PEFT： 在微调过程中使参数的子集可训练。
    • Reparameterized PEFT：为训练构建原始模型参数的（低维）重新参数化，然后等效地将其转换回推断。
    • Hybrid PEFT： 将不同PEFT方法的优势结合起来构建统一的PEFT模型。

    PEFT方法详解

    In-context Learning

    In-context learning 也叫 few-shot learning（少样本学习）。在推理阶段，给出下游任务的几个示例，就可以“激发”模型在这类任务上的能力。它的本质就是给予模型一些先验知识（prompt），模型输出 token 的概率分布就会改变（条件概率），进而得到合适的输出。

    Additive PEFT

    • Adapter类：通过在Transformer块内添加小型Adapter层，实现了参数高效微调
    • Soft Prompt类：通过在输入序列开头添加可学习的向量，以实现参数高效微调。代表性方法包括Prefix-tuning和Xprompt。
    • Others：SSF和IPA等方法也通过在模型中添加可学习参数来实现参数高效微调。这些方法通常将可学习的参数添加到Transformer块的某些操作（如键、值、查询矩阵、线性层等）中，以实现对模型表示的微调。

    Adapter

    论文：Parameter-Efficient Transfer Learning for NLP paper

    在面对特定的下游任务时，如果进行 Full-fintuning（即预训练模型中的所有参数都进行微调），太过低效；而如果采用固定预训练模型的某些层，只微调接近下游任务的那几层参数，又难以达到较好的效果。于是通过设计一个 Adapter 结构，将其嵌入 Transformer 的结构里面，在训练时，固定住原来预训练模型的参数不变，只对新增的 Adapter 结构进行微调。

    Adapter 结构：首先是一个 down-project 层将高维度特征映射到低维特征，然后过一个非线形层（常用GLUE）之后，再用一个 up-project 结构将低维特征映射回原来的高维特征；同时也设计了 skip-connection 结构，确保了在最差的情况下能够退化为 identity。

    Prefix-Tuning

    论文：Prefix-Tuning: Optimizing Continuous Prompts for Generation paper

    与 Full-finetuning 更新所有参数的方式不同，Prefix Tuning 方法是在输入 token 之前构造一段任务相关的 virtual tokens 作为 Prefix，训练的时候只更新 Prefix 部分的参数，而 Transformer 中的其他部分参数固定。该方法其实和构造 Prompt 类似，只是 Prompt 是人为构造的“显式”的提示，并且无法学习，而 Prefix 则是可以学习的“隐式”的提示。

    为了增强表达力，不只是嵌入层后的第一个 Transformer block，Prefix-tuning 对模型的每个 Transformer block 的输入都加入了 Prefix vector。同时，为了防止直接更新 Prefix 的参数导致训练不稳定的情况，他们在 Prefix 层前面加了 MLP 结构（相当于将 Prefix 分解为更小维度的 Input 与 MLP 的组合后输出的结果），训练完成后，只保留 Prefix 的参数。

    Prompt-Tuning

    论文：The Power of Scale for Parameter-Efficient Prompt Tuning paper

    Prompt-Tuning 是 Prefix Tuning 的简化版本。即只在输入层加入 prompt tokens，也不需要加入 MLP 进行调整来解决难训练的问题。同时，实验说明随着预训练模型参数量的增加，Prompt Tuning 的方法会逼近 Fine-tune 的结果。总而言之，只要模型规模够大，简单加入 Prompt tokens 进行微调，就能取得很好的效果。

    P-Tuning

    P-Tuning v1

    论文：GPT Understands, Too paper

    大模型的 Prompt 构造方式严重影响下游任务的效果，P-Tuning 提出将 Prompt 转换为可以学习的 Embedding 层，但直接对 Embedding 参数进行优化会存在这样两个挑战

    • Discretenes：对输入正常语料的 Embedding 层已经经过预训练，而如果直接对输入的 prompt embedding 进行随机初始化训练，容易陷入局部最优。
    • Association：没法捕捉到 prompt embedding 之间的相关关系。

    所以，P-Tuning 用 MLP+LSTM 的方式来对 prompt embedding 进行一层处理，从而获取prompt embeddings 之间的联系。同时，P-Tuning 方法在 context information (x)的 embedding e(x)，以及 target tokens (y) 的 embedding e(y)，都可以在前面拼接 pseudo prompts 的 embedding。

    P-Tuning v2

    论文：P-Tuning v2: Prompt Tuning Can Be Comparable to Fine-tuning Universally Across Scales and Tasks paper

    P-Tuning v2 试图让 Prompt Tuning 能够在不同参数规模的预训练模型、针对不同下游任务的结果上都达到 Fine-tuning 的结果。其方法是在多层加入了 Prompts tokens 作为输入，带来两个方面的好处：1)带来更多可学习的参数。2)加入到更深层结构中的 Prompt 能给模型预测带来更直接的影响。

    P-Tuning v2 实验分析

    • 用 MLP 重构参数并不能保证在所有下游数据集上有效；
    • 比较难的任务需要更长的 prefix prompt；
    • 可以采用多任务学习：在多个任务上一起训练共同的 prefix prompt vectors，然后在各个任务上分别微调。

    Selective PEFT

    Unstructured Masking方法

    通过在模型参数上添加可学习的掩码来确定可微调的参数。典型的工作包括Diff pruning、PaFi和FishMask。这些方法使用可学习的掩码来选择模型参数中的一部分进行微调，而保持其余参数固定。

    • Diff pruning：使用可学习的二值掩码来选择模型参数中改变最大的一部分进行微调。
    • PaFi：选择模型参数中绝对值最小的一部分进行微调。
    • FishMask：使用近似Fisher信息来选择模型参数中重要性最大的一部分进行微调。

    Structured Masking方法

    通过添加可学习的权重或偏差参数来确定可微调的参数。典型的工作包括S-Diff pruning、Bitfit和Xattn Tuning。这些方法通过添加可学习的权重或偏差参数来创建结构化的掩码，从而确定模型参数中的一部分进行微调。

    • S-Diff pruning：将Transformer块的权重参数分为多个节点，并使用L1正则化来确定重要性最高的节点进行微调。
    • Bitfit：只微调模型每个层中的偏差参数。
    • Xattn Tuning：只微调模型注意力层中的交叉注意力参数。

    Reparameterized PEFT

    Low-rank Decomposition方法：通过参数分解来构建低秩表示。典型的方法包括Intrinsic SAID、LoRA、Compacter和KronA。这些方法通过参数分解将原始模型参数分解为低秩矩阵和残差矩阵，从而引入可学习的参数。

    LoRA Derivatives方法：通过动态选择LoRA的秩来构建低秩表示。典型的方法包括DyLoRA、AdaLoRA和SoRA。这些方法通过动态选择LoRA的秩来构建原始模型参数的低秩表示，并添加额外的低秩矩阵来表示残差。

    LoRA

    论文：LoRA: Low-Rank Adaptation of Large Language Models paper

    一些研究对语言模型的参数进行研究发现：语言模型虽然参数众多，但是起到关键作用的还是其中低秩的本质维度。而微调的本质就是更新参数，LoRa(Low-Rank Adaption) 利用两个低秩矩阵替代某个大矩阵的思路，用两个 MLP 去近似参数更新量。

    LoRA 微调时，原来模型参数冻结，只更新 A 和 B 的参数。其中 r 称为 rank，是一个重要的超参数。 r越大，可训练参数越多，表达力越强，不过一般并不需要太大， r = 4,8,16 是常见的设置。此外，在 Transformer block 中，对参数矩阵的更新量进行模拟，一般是选择自注意力层的 Q, V 矩阵；范围更大时则选择 Q, K, V, O（输出矩阵）。

    基于大模型的内在低秩特性，增加旁路矩阵来模拟全模型参数微调，LoRA通过简单有效的方案来达成轻量微调的目的。GPT的本质是对训练数据的有效压缩，从而发现数据内部的逻辑与联系，LoRA的思想与之有相通之处，模型虽大，但起核心作用的参数是低秩的，通过增加旁路，达到很好的效果。

    DORA

    论文：DoRA: Weight-Decomposed Low-Rank Adaptation paper

    研究动机

    LoRA 可以被视为全微调的一种一般近似。通过逐渐增加 LoRA 的秩 r 以与预训练权重的秩一致，LoRA 可以达到类似于 FT 的表达水平。DoRA将权重矩阵重构为两个独立的分量，幅度和方向，以揭示 LoRA 和 FT 学习模式中的内在差异。 其中 是幅度向量，是方向矩阵，是矩阵在每列上的向量范数。

    Unified View of PEFT

    论文：Towards a Unified View of Parameter-Efficient Transfer Learning paper

    方法回顾

    • Adapter是把input representation经过矩阵W_down映射到低维空间，然后经过非线性激活层，比如ReLU，然后再又经过矩阵W_up映射回高维空间。

    • Prefix tuning 把每一层attention的key和value前面都prepend一些向量，把attention原本的K和V前面分别加上P_k,P_v，得到新的attention。

    • LoRA用两个low rank的矩阵的乘积近似估计原本W_q和W_k这两个矩阵的参数更新情况。

    统一框架

    在此基础上，通过对比分析了Adapter,Prefix Tuning和LoRA这三种design的具体结构，提出了一个通用框架。将其均视为学习hidden representations的修改向量 Δh。具体来说，h 为原始未修改的 hidden representations，x 为 PLM 子模块的输入（即 h 和 x 分别是注意力输出和输入）。定义了以下四个设计维度，并且可以通过沿这些维度的不同值来实例化不同的方法。

    • functional form：指计算 Δh的函数形式。其形式均为 proj down → nonlinear → projup 架构。
    • insertion form：指functional form怎么和原来的部分连接，像adapters就是从h出去的，作者把它形容为串联；而Prefix Tuning和LoRA都是从x出去的，作者把它们形容为并联。
    • Modified representation：指这些设计是作用于Transformer的具体位置。像prefix tuning是作用在attn上的；adapter同时作用在attn和ffn上；LoRA是作用在attn上的。
    • composition function：表示Δh和h如何结合。

    在此统一框架下，通过组合不同的元素，可以得到一下新的方法：

    • Parallel Adapter：把adapter的insertion form改成并联的结构。
    • Multi-head Parallel Adapter：把并联的adapter用在每一个head上面。与Prefix tuning十分接近了（原本的Adapter是single-head）
    • Scaled Parallel Adapter：并联的Adapter后面加个常数，和LoRA类似。
    实验分析
    • 对于insertion form，无论是加在attn还是加在ffn上，Parallel Adapter都比Sequential Adapter更好
    • 对于modified representation，加在feed forward上效果更好。值得注意的是：prefix tuning有天然的弊端，只能作用在attn上，而不能作用在效果更好的ffn上面，prefix tuning比起Adapter和LoRA不够灵活。
    • multi-head在参数量很少和参数数量很多时基本一致；而作用在ffn上的设计受参数量影响较大。因此，只想改动少量参数时，用multi-head结构； 而调整较大参数的时候，可以用ffn结构。

    总结与未来方向

    总结

    • PEFT技术通过最小化额外参数和计算资源的引入，实现了大型模型在下游任务中的高效微调。在自然语言处理、计算机视觉等多个领域取得了显著的性能提升，同时大幅减少了计算成本。
    • 不同类型的PEFT方法适用于不同的下游任务和模型规模，需要根据具体任务和模型需求选择合适的PEFT方法。

    未来方向

    • PEFT方法通常对超参数敏感，手动调优需要大量努力。开发不依赖手动调优的PEFT方法，或者自动寻找最优的超参数设置。
    • 增强训练效率：PEFT方法在计算效率上存在提升空间，尤其是训练阶段的内存和计算资源需求。探索优化内存和计算效率的方法，如模型压缩技术。
    • 探索扩展定律：PEFT方法在较小模型上的效果不一定适用于大型模型。探索如何适应大型模型的PEFT方法。
